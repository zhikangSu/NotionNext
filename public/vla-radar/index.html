哪些真的变了？

主流 VLA 的大变化

  • 🧠
    Backbone 变了：从机器人专用 Transformer，转向预训练 VLM / LLM 作为语义与感知骨干。
  • 🦾
    Action decoder 变了：从离散 action token，转向 flow / diffusion action expert 或 token+flow hybrid。
  • 🧭
    Prompt 变了：不再只是一句任务指令，而是 subtask、goal image、metadata、history、control mode。
  • 🔁
    数据飞轮变了：从成功演示扩展到 web data、人类视频、失败、自主 rollout、intervention、reward。
哪些没怎么变？

底层骨架仍然很稳

  • 🔷
    Transformer 仍是主干：变的是 token、attention、expert、context，不是基础范式。
  • 👁️
    输入核心没变：RGB + proprioception + language 仍是最常见组合。
  • 🎮
    输出仍是控制命令：joint、EEF、gripper、base velocity，然后接 PD / IK / chunk execution。
  • 📦
    机器人数据仍是瓶颈：web 语义能迁移，但物理交互、接触、灵巧动作仍靠机器人经验。

后训练怎么选？

全参微调 / LoRA / 冻结 VLM

三者本质是对原始 VLA “动刀深度”不同：越深越能适配新域，也越容易破坏原始语义与泛化。

全参微调：能力上限最高，破坏性最大

适合新 embodiment / 新相机 / 新动作空间差异很大，且你有足够多样数据。风险是灾难性遗忘、语言跟随下降、VLM 语义空间被低层动作 loss 拉坏。

LoRA：原权重不动，给模型外挂任务偏置

适合目标机器人接近预训练分布、数据量中小、希望保留 base 的通用能力。实践上常用 VLM/LLM LoRA + action expert/head 全参训练。

冻结 VLM：最保护语义，最适合稳定复现

把 VLM 当固定视觉语言特征库，只训练 action expert / projector。优点是小数据稳、显存低、语义不被破坏；缺点是固定特征若缺少几何/接触信息，上限会受限。

推荐路线：先冻结，再 LoRA，最后局部/全参

SmolVLA 复现可以先冻结 VLM 验证数据与 action expert；再给后半层或 projector 加 LoRA；只有在有 replay、OOD 验证和足够多样数据时才做局部/全参。对 RL 后训练，尽量隔离高方差动作梯度。
